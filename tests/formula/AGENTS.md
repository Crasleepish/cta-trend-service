# tests/formula

## 作用与架构定位

* 职责：白皮书公式一致性测试（最小样本 → 明确期望）。
* DOT 位置：Algorithm Tests / Formula Test Suite。

## 目录结构（至多两层）

```
.
`-- test_wp*_*.py
```

## 本模块约束/规范

* 文件名必须包含白皮书章节号（如 `test_wp4_trend.py`）。
* 每个测试文件首行注释必须包含 `# Ref: WP §x.x`。
* 允许在测试内写公式与数值期望；业务代码中不展开白皮书细节。
* xfail 仅在章节未实现且已明确批准时使用；当前公式用例应为真实断言。
